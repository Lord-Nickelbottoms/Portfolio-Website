Agrifund

An iOS native application for small scale farmers to assist with getting funding and build up a portfolio to track profits/loss that can be used for business funding. The app communicated with a Java Springboot backend for CRUD operations.

This massive project allowed us to apply what we have learned on iOS development, the Swift programming language and UIKit. From UI design and constraints, to networking using Swift and Firebase, the challenge was immense but not impossible.

_Surveys

An iPadOS application born out of a job application assessment. The task required a UI that users could interact with a survey, the data would then need to be saved to a database. The application would have to retrieve the data out of the database and perform operations or calculations on it and display it on a Results page.

Because UIKit doesn't have a native way of creating multi-column tables at the time, I had to create one manually and position the columns and rows to appear as if it were a table. I would then code the necessary Outlets and functions to get the information from Pseudo-radio buttons. The UI itself was the biggest challenge compared to the actual requirement of the task. The database used was CoreData.
